## 3. Pauta e Discussões

### Tópicos Abordados

- *Finalização do Sketch*
Foi feito o Rich Picture da ideia central do jogo e a definição de como seriam os elementos de UI para o protótipo de média fidelidade, como o menu principal, a tela de jogo e a tela de resultados.

- *Decision*
Com a ideia do que seria o jogo já clara entre os membros, foi possível definir a direção do projeto, o que inclui a definição de uma estratégia de desenvolvimento ágil, com entregas parciais e incrementais, para melhorar o controle do progresso e manter alinhamento contínuo com as definições da fase. Optamos por escolher o modelo de cascata incremental, onde cada etapa do processo de desenvolvimento é realizada de forma sequencial, mas com entregas parciais e incrementais, para melhorar o controle do progresso e manter alinhamento contínuo com as definições da fase.
Discutimos sobre a capacidade de desenvolvimento da equipe, alinhamos expectativas e duvidas sobre o desenvolvimento em si de um jogo eletrônico, foram apresentados dois jogos na reunião de acervo pessoal do Mateus para apresentar referência do estilo visual que seria seguido para confeccionar o protótipo de média fidelidade.
Foi conversado também sobre o possivel tamanho do projeto considerando linhas de código, informação necessária para a confecção do documento de estimativas.

- *Repositório do Projeto*
Foi aprovada a estrutura do repositório do projeto, onde cada artefato terá uma pasta específica para armazenar os arquivos relacionados a ele, e também foi aprovada a estrutura de branches do projeto, onde cada membro do grupo terá uma branch específica para desenvolver suas tarefas, e a branch principal será utilizada para integrar as entregas parciais e incrementais.

- *BPMN do Processo de Desenvolvimento*
Foi discutido sobre a modelagem do processo de desenvolvimento do projeto, e foi decidido que cada membro do grupo irá modelar uma parte do processo, e depois será feita uma integração das partes para formar o BPMN completo do processo de desenvolvimento.
